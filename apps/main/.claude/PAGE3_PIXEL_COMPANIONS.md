# 页面 3：像素伙伴展示柜

## 页面概念
这是一个复古的街机展示柜，里面陈列着你最珍视的游戏角色——它们以 2D 像素形态活过来，各自演绎着经典的待机动画。用户仿佛置身于游戏厅，透过玻璃柜欣赏这些陪伴你成长的“伙伴”。

## 布局与视觉
-   **背景**：深色木质纹理或拉丝金属板，带有霓虹灯管勾勒的轮廓。整体氛围怀旧而温暖。
-   **展示柜结构**：屏幕划分为 3x3 或 3x4 的格子，每个格子是一个独立的“展示窗”。展示窗有凸起的边框、内发光，仿佛一个个小舞台。
-   **角色陈列**：每个展示窗内有一个像素角色（例如：林克、卡比、亚瑟 Morgan、Pac-Man 等），尺寸约为 64x64 或 96x96 像素。角色下方显示其名字和出自的游戏（像素字体）。

## 核心动效：角色动画
-   **待机动作**：每个角色循环播放简单的像素动画，例如：
    -   林克：轻微上下浮动，偶尔拔剑又收回。
    -   卡比：膨胀收缩，左右摇摆。
    -   亚瑟：挥舞小刀或抖动手臂。
    -   Pac-Man：张嘴闭嘴，原地旋转。
-   **动作风格**：帧数较低（6-12 fps），刻意保留像素动画的顿挫感。动作幅度小但传神。

## 交互反馈
-   **鼠标悬停**：
    -   对应的展示窗边框变为霓虹紫色并轻微发光。
    -   角色停止待机动作，播放一个特殊的“打招呼”动画（如跳跃、挥手、眨眼）。
-   **点击角色**：
    -   展示窗背景短暂闪烁白色。
    -   弹出浮动对话框（像素风格的气泡），里面显示一段你对这个角色的简短回忆或一句经典台词。对话框显示数秒后自动消失。
    -   可选：伴随 8-bit 风格的音效（如“叮”的一声）。

## 技术实现建议
-   使用 **Canvas** 绘制所有角色和背景，方便逐帧动画控制和像素风格渲染。
-   每个角色维护一个状态机（待机、悬停、点击），根据鼠标坐标（需进行坐标映射）切换动画。
-   动画使用 `requestAnimationFrame` 驱动，每帧更新画布。可预加载所有角色的精灵表（spritesheet）。
-   如果角色数量不多，也可以考虑用多个 `<canvas>` 元素分别绘制每个角色，降低重绘复杂度。

## 代码结构要求
-   JavaScript 模块化：一个角色类（`PixelCompanion`）管理位置、动画帧、状态。
-   主循环遍历所有角色并调用其 `draw` 方法。
-   鼠标事件监听需转换为画布内的格子索引。
