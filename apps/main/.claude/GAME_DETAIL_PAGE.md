# 游戏详情页设计说明

## 一、页面概述

游戏详情页是独立于主滚动页面的子页面，通过路由 `/game/:id` 访问。用户从第2页（落叶卡牌殿堂）点击卡牌上的“进入详情”按钮后跳转至此，浏览选定游戏的专属图集。页面设计强调沉浸式体验，每次进入随机展示三种模式之一，且保证不会连续两次看到相同模式。

## 二、图片资源与数据

-   **图片存放**：每个游戏的图集图片位于 `public/assets/images/games/[gameId]/` 目录下，图片按数字序号命名（`1.jpg`, `2.jpg`, `3.jpg` ...）。游戏ID使用英文小写连字符，例如 `league-of-legends`、`red-dead-redemption-2`。
-   **图片数量配置**：由于前端无法动态读取目录，需在代码中预定义每个游戏的图片数量映射表，例如：
    
    ```javascript
    const GAME_IMAGE_COUNTS = {
      'league-of-legends': 8,
      'red-dead-redemption-2': 6,
      'monster-hunter': 10,
      // ...
    };

-  **路由参数**：`id` 即为游戏ID，组件内根据ID加载对应图片列表

## 三、三种展示模式

每次进入详情页，从以下三种模式中随机选择一种呈现，且保证当前游戏不会连续两次出现相同模式（使用 `localStorage` 记录上次模式）。

### 模式 A：相册模式

- **视觉风格**：经典翻页相册，中央显示当前图片，左右两侧有半透明箭头按钮。下方显示缩略图条或页码指示器。
- **交互细节**：
  - 点击左右箭头或按键盘左右方向键，切换图片，切换动画为平滑的滑动翻页（可使用GSAP的 `x` 过渡或CSS `transform`）。
  - 支持循环切换（最后一张向右切到第一张，反之亦然）。
  - 点击缩略图可直接跳转到对应图片。
  - 右上角提供“返回”按钮，点击返回上一页（主页面）。
- **技术实现**：
  - 使用 Solid 的 `createSignal` 存储当前索引 `currentIndex`。
  - 切换时用 GSAP 对图片容器做位移动画，同时更新索引。
  - 缩略图区域用 Flex 展示小图，点击时更新索引并触发动画。

### 模式 B：3D 轮播卡牌模式

- **视觉风格**：卡片围绕垂直 Y 轴排列成一个圆环，所有卡片面向圆心，类似旋转木马。每张卡片正面显示一张图集中的图片。
- **交互细节**：
  - **自动步进转动**：轮播会自动一格一格转动（非连续转动），每次转动一个卡位的角度（例如有 N 张卡片，每次转动 `360/N` 度）。转动间隔固定为 3 秒。转动动画使用 GSAP 的 `rotateY` 补间，带有缓动效果。
  - **点击卡片**：点击任意卡片时，该卡片以“抽卡”动画效果（如飞出、放大）移动到屏幕中央，放大展示图片。其他卡片淡出或隐藏。被选中的卡片变为大图模式。
  - **退出详情**：大图模式时，卡片右上角显示一个“退出游戏”按钮（或返回图标），点击后大图以反向动画回到轮播中的原位，恢复轮播状态（包括自动转动）。
  - **手动切换**：在大图模式下，可增加左右箭头切换图片（类似相册模式），但此时轮播被暂停。
- **技术实现**：
  - 使用 CSS 3D 变换布局卡片：容器设置 `transform-style: preserve-3d`，每张卡片通过 `rotateY(angle)` 和 `translateZ(radius)` 定位在圆环上。
  - 自动转动：通过 `setInterval` 更新 `rotation` 信号，并将该值应用到容器（`rotateY(rotation)`）。注意卡片本身不需要旋转，只需容器旋转，但为了让卡片始终面向圆心，卡片本身需要反向旋转？这里需要计算：如果容器旋转 `θ`，则卡片应设置 `rotateY(-θ)` 以保持面向圆心。但为了简化，可以让卡片不面向圆心，而是面向外，也可以接受。但用户要求“卡片围着y轴形成一个圆(跑马灯效果)”，通常跑马灯卡片面向外。我们按跑马灯通常实现：卡片面向外，容器旋转即可。但点击卡片时可能需要单独处理。
  - 点击卡片：获取被点击卡片的索引，计算其当前位置，然后通过GSAP将该卡片从轮播中移除（临时设为绝对定位），并移动到屏幕中央，同时放大。其他卡片透明度设为0。
  - 退出：将大图卡片通过GSAP移回原位，恢复所有卡片透明度，重新启用自动转动。
  - “退出游戏”按钮需要在大图卡片上显示，点击时触发退出动画。

### 模式 C：抽签盒子模式

- **视觉风格**：一个古风抽签盒子（可用SVG或Canvas绘制），盒子旁有一个签筒或盒子本身作为抽签入口。签面上带有裁剪后的图片局部。
- **交互细节**：
  - **点击盒子抽签**：点击盒子，盒子播放抖动动画（快速左右晃动+旋转），同时从盒子中“吐”出一个签，落在盒子前方。签面上显示当前图片的随机裁剪区域（可使用Canvas裁剪图片的一部分）。
  - **查看完整图片**：点击签，签以画轴展开动画（从中心向两边展开，或从卷曲到平铺）展示完整图片。展开后图片占据屏幕大部分区域。
  - **关闭展开**：点击展开后的图片任意位置或外部区域，图片卷回收缩为签，回到盒子旁。
  - **再次抽签**：再次点击盒子，抽下一张图片（按图集顺序循环），更新签面为新图片的随机裁剪区域。
- **技术实现**：
  - 盒子抖动：GSAP快速改变 `x` 和 `rotation` 值，形成震动效果。
  - 签的绘制：可使用一个 `<canvas>` 绘制签的形状和裁剪的图片。或者使用两个 `<div>` 分别表示签的上半部和下半部，中间用背景图片裁剪。推荐使用Canvas以精确控制裁剪区域。
  - 画轴展开动画：可使用两个卷轴元素（左侧卷轴、右侧卷轴）从中间向两侧移动，逐渐揭示完整图片。或者使用 GSAP 对图片容器的 `scaleX` 或 `clip-path` 进行动画。简单起见，可以用一个图片容器，初始宽度为0，然后动画到100%，同时配合两个卷轴图片移动。
  - 签面裁剪：每次抽签时，随机选择图片区域（例如随机x,y偏移，固定裁剪宽度和高度），用Canvas绘制到签面上。
  - 抽签后，当前图片索引循环到下一张。

## 四、随机模式交替逻辑

- **存储键**：`lastMode_${gameId}` 存储在 `localStorage` 中，值为上次使用的模式名称（`'album'`, `'carousel'`, `'lottery'`）。
- **随机算法**：
  1. 定义模式数组 `['album', 'carousel', 'lottery']`。
  2. 读取 `lastMode`，如果不存在则从数组中随机选择一个。
  3. 如果存在，则从数组中过滤掉 `lastMode`，从剩余两个中随机选择一个。
  4. 更新 `localStorage` 为本次选择的模式。
- **实现位置**：在 `GameDetail` 组件的 `onMount` 或首次渲染时执行。

## 五、返回与导航

- 详情页应始终包含一个返回按钮（左上角或右上角），点击后调用 `navigate(-1)` 返回上一页（即主页面中的第2页）。也可直接返回首页，但用户体验上应该回到之前的位置，所以 `-1` 最合适。
- 在3D轮播的大图模式下，“退出游戏”按钮只负责返回轮播状态，并非返回主页；返回主页的按钮应独立存在。

## 六、技术要点总结

- **框架**：Solid.js 组件，使用 `useParams` 获取 `id`。
- **动画**：GSAP 用于所有复杂动画（抖动、转动、飞出、展开）。
- **图片加载**：使用 `<img>` 标签或预加载所有图片至内存，以便Canvas绘制。
- **响应式**：确保在桌面端大屏上布局合理，图片大小适应视口。
- **模式切换**：根据随机结果，在JSX中通过 `<Switch>` 或条件渲染展示对应组件（`<AlbumMode />`, `<CarouselMode />`, `<LotteryMode />`），便于维护。
