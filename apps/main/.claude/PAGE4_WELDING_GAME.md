# 页面 4：焊接中的未来游戏

## 页面概念
这是整个网站的终章，象征你从玩家转变为创造者。画面是一个工作台，上面放着一块平板电脑（或类似画板的设备），平板屏幕内正运行着一个简单的动作游戏——一个像素风格的主角正在与怪物战斗，循环往复。平板旁有两只手，各自握着一支电焊笔。你可以点击屏幕任意位置，手会移动电焊笔到该处，并迸发出像素火花，仿佛你在亲手焊接、修改这个游戏世界。

## 布局与视觉
-   **背景**：木纹工作台，散落着几颗螺丝、一张图纸。采用暖色滤镜，营造工作室氛围。
-   **平板屏幕**：占据画面中央较大区域，有厚实的边框。屏幕分辨率为 400x300 左右，内部是像素游戏画面。
    -   **游戏内容**：一个像素小人（主角）在左侧，一个简单的史莱姆状怪物在右侧。主角循环播放攻击动画（如挥剑），怪物循环播放受伤/待机动画。背景有简单的云或地面线条。
-   **双手与电焊笔**：屏幕两侧（或下方）各有一只像素风格的手，握着电焊笔。手本身也有简单动画（手指轻微颤动）。电焊笔尖端有一个微小的光点。

## 核心动效与交互
1.  **屏幕内游戏循环**：
    -   使用 Canvas 绘制游戏循环。主角和怪物持续播放攻击/受击动画，可加入简单的生命值条（或不加）。
2.  **点击交互**：
    -   用户点击屏幕内的任意坐标。
    -   手（根据点击位置在屏幕左侧或右侧，选择最近的一只）带着电焊笔平滑移动到点击点。移动过程中，电焊笔末端拖曳微弱轨迹。
    -   到达目标点后，播放 **焊接火花效果**：
        -   火花为粒子效果：多个亮黄色/白色的小像素点从点击处向外溅射，然后快速消失。
        -   同时点击处可能出现一个小的“焊接点”（一个亮斑），并慢慢变暗。
    -   如果点击在主角或怪物身上，可触发额外的反应（如主角闪烁、怪物颤抖），表示正在“修改”它们。
3.  **手部动画**：
    -   手在移动时，电焊笔的角度略微变化，跟随移动方向。
    -   闲置时，手有轻微的上下浮动（呼吸感）。

## 技术实现建议
-   整个页面使用一个 **主 Canvas** 绘制，分层管理：
    -   背景层（工作台、平板边框）
    -   游戏层（主角、怪物、背景）
    -   手部层（两只手和电焊笔）
    -   火花层（粒子效果）
-   游戏角色动画也用 Canvas 绘制，维护简单的帧计数器。
-   点击处理：监听 Canvas 的点击事件，将屏幕坐标转换为 Canvas 坐标，判断是否在平板区域内。如果在，触发手部移动和火花效果。
-   手部移动可使用缓动函数（如 `easeOutCubic`）实现平滑移动。
-   火花粒子：每个粒子有位置、速度、寿命，每帧更新并重绘。

## 代码结构要求
-   定义 `Hand` 类，管理位置、目标位置、移动状态。
-   定义 `Spark` 类，用于粒子系统。
-   游戏循环统一由 `requestAnimationFrame` 驱动，每帧清空画布并按顺序重绘所有层。
-   所有元素保持像素风，可使用 `imageSmoothingEnabled = false` 避免抗锯齿。
