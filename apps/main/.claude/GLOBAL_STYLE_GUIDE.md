

# 全局风格指南

本文档定义了项目的视觉语言和设计系统。AI在生成所有页面代码时，必须严格遵守此规范，确保整体风格统一、连贯，且符合“国外设计感拉满的动画风格”。

## 1. 设计哲学

-   **动画优先（Animation-First）**：整个网站应该像一部可交互的动画短片。动效不仅是点缀，而是叙事和交互的核心。所有元素都应有“生命力”——即使静止，也暗示着运动的可能性。
-   **手绘线条感（Hand-drawn Aesthetic）**：界面元素（边框、图标、装饰）应具有类似钢笔或马克笔勾勒的线条感，略带不规则、有弹性的边缘，避免生硬的纯几何图形。可以使用 SVG 描边或 CSS 的 `border` 配合 `border-radius` 和自定义的 `box-shadow` 来模拟。
-   **饱和色彩与强烈对比**：色彩饱和度高，明快有力。背景通常较深，以突出前景的霓虹色和亮色。避免使用 muted 或 pastel 色调，除非用于特定的怀旧效果（如小霸王黑白灰）。
-   **游戏化反馈**：每一个可交互元素（按钮、卡片、按键）的反馈都必须夸张、有趣，带有“漫画感”——例如点击时的放大脉冲、弹跳、光效或拟声词文字。
-   **层次感与景深**：通过阴影、模糊、缩放和动画营造出明确的空间层次，让用户感觉界面是多层的，而非扁平。

## 2. 色彩系统

使用 CSS 自定义变量定义所有颜色，确保全局一致性。

```css
:root {
    /* 背景色 - 深邃的游戏空间 */
    --bg-deep: #0B1026;        /* 深空蓝，主背景 */
    --bg-surface: #1A1F35;      /* 稍亮，用于卡片、面板的底层 */
    --bg-overlay: rgba(10, 15, 30, 0.8); /* 半透明遮罩 */

    /* 主色 - 霓虹科技感 */
    --primary-neon: #8A2BE2;     /* 霓虹紫，用于线条、高亮、图标 */
    --primary-glow: #4169E1;     /* 电光蓝，用于光效、次要高亮 */
    --primary-accent: #FFD700;    /* 警示黄，用于按钮悬停、特效文字 */
    --primary-energy: #FF4500;    /* 活力橙，用于紧急反馈、打击特效 */

    /* 功能色 - 文本与信息 */
    --text-light: #F0F0F0;       /* 柔白，主要文字 */
    --text-muted: #A9A9A9;       /* 冷灰，辅助文字、时间线点 */
    --text-dark: #1E1E1E;        /* 深色背景上的文字（极少用） */

    /* 卡片边框色 - 动态变化，此为默认备用 */
    --card-border-default: rgba(138, 43, 226, 0.6); /* 霓虹紫半透明 */

    /* 像素风专用色 - 第3页使用 */
    --pixel-shadow: #2C1810;      /* 深褐色，用于像素角色阴影 */
    --pixel-skin: #FFDAB9;        /*  peachpuff，像素角色肤色 */
}
```

- **应用原则**：

  - 背景大面积使用 `--bg-deep`。
  - 重要线条和高亮使用 `--primary-neon` 或 `--primary-glow`。
  - 交互反馈（悬停、点击）使用 `--primary-accent` 或 `--primary-energy`。
  - 文字信息使用 `--text-light` 和 `--text-muted`。

  ## 3. 排版

  css

  ```
  :root {
      /* 标题字体：漫画风格，有冲击力 */
      --font-heading: 'Bangers', system-ui, cursive;
      /* 正文字体：现代、科技感、可读性强 */
      --font-body: 'Space Grotesk', sans-serif;
      /* 像素字体：用于第3、4页的像素元素 */
      --font-pixel: '"Press Start 2P"', cursive, monospace; /* 或 VT323，更复古 */
  }
  ```

  

  - **标题**：全大写，字距宽松，使用 `--font-heading`。用于页面大标题、设备名称、游戏标题等。
  - **正文**：常规大小，清晰易读，使用 `--font-body`。
  - **像素文字**：极小号（12-14px），用于像素角色标签、对话框、焊接屏幕内的文本，使用 `--font-pixel`。注意 `Press Start 2P` 较大，可能需要缩小字号。
  - **行高**：标题通常 1.2，正文 1.5。
  - **字体加载**：在 `index.html` 中引入 Google Fonts 或使用 fallback。

  ## 4. 核心组件样式

  ### 4.1 按钮与按键

  - **形状**：矩形，圆角适中（`border-radius: 12px` 左右）。可用微小的内凹阴影模拟按下效果。
  - **默认**：填充色 `--primary-neon` 或半透明背景，文字为 `--text-light`。
  - **悬停**：背景变为 `--primary-accent`，同时有放大脉冲动画（`scale: 1.05`），阴影扩散。
  - **点击（按下）**：动画：瞬间缩小（`scale: 0.95`），同时产生一个扩散的圆形光晕（`box-shadow: 0 0 20px var(--primary-energy)`），并在附近出现一个漫画拟声词（如“POW！”、“CLICK！”），拟声词随机选择，使用 `--font-heading`，绝对定位，动画结束后消失。
  - **键盘键（第1页设备上的按键）**：绘制为凸起的矩形（模拟老式按键），点击时同样有按下动画（下沉+弹起），并伴有光效。

  ### 4.2 卡片

  - **边框**：2px 实线，默认使用 `--card-border-default`，但每个卡片应根据游戏主题动态设置边框颜色（如《荒野大镖客2》深红色）。边框应具有轻微的内发光（`box-shadow: inset 0 0 5px currentColor`）。
  - **背景**：半透明的深色背景 `--bg-overlay` 或 `--bg-surface`，使背景图或下层元素隐约可见。
  - **圆角**：较大，`border-radius: 20px` 或更高，符合动画风格的可爱感。
  - **阴影**：外发光或投影，颜色为边框色的半透明版本。
  - **内容**：包含图片（或海报）、游戏名称、年份、简短描述。文字使用 `--text-light`，游戏名称用 `--font-heading` 小号。
  - **悬停**：轻微上浮，边框亮度增加，阴影放大，图片有微小的缩放。

  ### 4.3 时间线节点

  - **普通年份点**：直径为 8px 的圆点，背景色 `--text-muted`，无交互反馈。
  - **高亮年份点**：直径 12px，背景色 `--primary-neon`，带有呼吸灯动画（`box-shadow` 脉动）。鼠标悬停时，圆点周围出现一个旋转的光环。
  - **连线**：时间线本身的线条为 1px 虚线，颜色 `--text-muted`。

  ### 4.4 图标与装饰

  - 大量使用几何图形（圆、方、三角、星形）作为背景装饰，用 `--primary-neon` 或 `--primary-glow` 半透明绘制。
  - 可随机绘制一些看似随意的手绘线条（如波浪线、螺旋线），增加手绘感。

  ## 5. 动画核心原则

  - **缓动函数**：优先使用弹性缓动，例如 `cubic-bezier(0.68, -0.55, 0.27, 1.55)`（产生弹跳效果）或 `cubic-bezier(0.25, 0.46, 0.45, 0.94)`（标准减速）。避免线性动画。
  - **动效层级**：复杂动画应分层进行，有先有后，营造节奏感。例如卡片展开时，先放大，再显示内容。
  - **反馈速度**：悬停反馈应在 0.1s 内开始；点击反馈应瞬时（<0.05s）并迅速恢复。
  - **粒子效果**：火花、光效等粒子应使用 Canvas 绘制，每帧更新，并带有缓动（如粒子速度逐渐降低）。
  - **滚动同步**：页面滚动时，可触发视差效果或动画进度（如第1页时间线随滚动移动），使用 GSAP ScrollTrigger 或 IntersectionObserver 实现。

  ## 6. 响应式设计要点

  - **桌面优先**：以 1440px 及以上宽度为设计基准。
  - **缩放**：所有尺寸使用 `rem` 或 `vw` 相对单位，确保在不同屏幕下比例协调。
  - **Canvas 自适应**：Canvas 尺寸随窗口变化而重新计算并重绘内容，保持内容比例不变或重新布局。
  - **隐藏非核心元素**：在小屏（<1024px）上，可简化部分装饰，但核心交互必须保留。

  ## 7. 代码实现约束

  - 所有颜色、字体、间距、圆角必须使用 CSS 变量，便于全局调整。
  - Tailwind 配置中应扩展这些变量作为颜色类和动画类。
  - GSAP 动画的持续时间、缓动应在常量中定义，便于统一修改。
  - Canvas 绘图时，颜色也要引用 CSS 变量（通过 `getComputedStyle` 获取），保持与 CSS 一致。

  ------

  **AI 特别注意**：

  - 任何出现在屏幕上的文字，如果带有特效（如打击特效），其字体应为 `--font-heading`，颜色为 `--primary-energy` 或 `--primary-accent`。
  - 所有按钮和可点击元素，除了视觉反馈，还应考虑焦点状态（:focus-visible），确保键盘可访问。
  - 第3页像素角色动画的帧率应控制在 8-12fps，以模仿老式游戏的顿挫感。
  - 第4页焊接火花粒子的颜色应为亮黄/白色，逐渐变暗消失。
