## 一、项目概述

-   **项目名称**：游戏时光博物馆
-   **核心主题**：以游戏陪伴我度过艰难时期为情感主线，通过四个章节展示游戏设备、经典游戏卡牌、像素角色以及我作为未来创造者的愿景。
-   **设计风格**：严格遵守 `GLOBAL_STYLE_GUIDE.md` 中定义的动画风格、色彩系统、排版和动画原则。
-   **技术栈**：
    -   **框架**：Solid.js（使用组件化开发）
    -   **样式**：Tailwind CSS（结合自定义动画类）
    -   **路由**：@solidjs/router（用于游戏详情页的独立路由）
    -   **动画**：GSAP 用于复杂时间线动画和缓动控制
    -   **绘图**：Canvas 用于大量粒子/卡牌渲染；SVG 用于静态图标和装饰
-   **页面构成**：共四个页面，从上到下垂直排列，通过平滑滚动切换（视差滚动效果）。
    -   第 1 页：游戏设备博物馆 (`PAGE1_CONSOLE_TIMELINE.md`)
    -   第 2 页：落叶卡牌殿堂 (`PAGE2_CARDS_FALL.md`)
    -   第 3 页：像素伙伴展示柜 (`PAGE3_PIXEL_COMPANIONS.md`)
    -   第 4 页：焊接中的未来游戏 (`PAGE4_WELDING_GAME.md`)

## 二、整体架构与技术要求

### 2.1 项目结构
-   使用 Solid.js 的组件化开发，每个页面为一个独立组件（如 `Page1Console`、`Page2Cards` 等）。
-   根组件 `App` 负责引入所有页面组件，并实现滚动视差容器（`h-screen snap-y snap-mandatory`）。
-   可选：使用 `@solidjs/router` 配置路由，但实际页面切换依然通过滚动完成（路由仅用于可能的深层链接或未来扩展）。

### 2.2 样式与动画

- **Tailwind CSS**：全局样式通过 `tailwind.config.js` 扩展主题颜色（与 `GLOBAL_STYLE_GUIDE.md` 保持一致），动画类使用 `@keyframes` 注册在 `tailwind.config.js` 的 `theme.extend.animation` 中。
- **GSAP**：
  - 用于第1页的设备切换动效、按键打击特效。
  - 用于第2页的卡牌落叶复合旋转（可创建 GSAP 时间线控制每个卡牌的随机运动）。
  - 用于第4页的手部移动和火花粒子效果。
  - 用于游戏详情页中3D轮播的卡片转动动画、抽签盒子的抖动和展开动画。
  - GSAP 实例应在组件 `onMount` 中创建，并在清理时注销。
- **Canvas 管理**：
  - 第2、3、4页使用 Canvas，通过 Solid 的 `ref` 获取 DOM 元素，在 `onMount` 中初始化绘图上下文。
  - 使用 `createEffect` 和 `createMemo` 管理响应式数据，并在 effect 中触发重绘。
  - 动画循环使用 `requestAnimationFrame`，并在清理时取消。

### 2.3 响应式与兼容性

- **基础响应式**：使用 Tailwind 的响应式工具类确保桌面端（1920x1080 及以上）和主流笔记本屏幕（1368x768）布局正常。移动端暂不深入优化，但需保证内容不溢出、可操作。
- **浏览器兼容**：针对最新版 Chrome、Firefox、Edge。

## 三、各页面实现要点（结合 Solid + GSAP）

### 第 1 页：游戏设备博物馆

- **组件**：`<Page1Console />`
- **时间线**：使用 Flex 生成点状节点。通过 Solid 的 `createSignal` 管理当前高亮年份。
- **设备切换**：
  - 设备图形使用 SVG 绘制（便于缩放和样式控制），或使用 HTML 元素配合 Tailwind 绘制。
  - 鼠标悬停于高亮年份时，触发 GSAP 动画：旧设备 `scale` + `rotate` 消失，新设备弹性出现。
- **按键特效**：每个按键（SVG 中的区域或独立元素）绑定 `onClick`，使用 GSAP 创建“按下”动画（`scale` 脉冲），并生成一个临时绝对定位的 `<div>` 显示“POW！”等特效文字，动画结束后移除。
- **其他年份点**：点击后触发父组件传递的滚动函数，滚动到第2页，并通过 Solid 的 Context 或 store 传递选中年份，第2页接收后高亮对应卡片。

### 第 2 页：落叶卡牌殿堂

- **组件**：`<Page2Cards />`
- **Canvas 实现**：
  - 在 `onMount` 中初始化 Canvas 和 GSAP 时间线。
  - 定义 `Card` 类（位置、速度、旋转、图片等）。每帧更新并重绘。
  - 卡片的生成：使用 `setInterval` 或 `requestAnimationFrame` 控制生成速率，最多 30 张。
- **点击交互**：
  - 监听 Canvas 的点击事件，转换为坐标，遍历卡牌检测拾取。
  - 被点击卡牌进入聚焦状态：GSAP 暂停所有卡牌动画，将该卡牌移至画布中心并放大（同时继续旋转），其他卡牌淡出。
  - 显示游戏详情：在 Canvas 上方叠加一个半透明浮层，显示游戏名称和一个“进入详情”按钮。点击按钮使用 `useNavigate` 跳转到 `/game/${gameId}`。
- **性能优化**：GSAP 仅用于主时间线控制，卡牌物理运动仍用 JS 计算。

### 第 3 页：像素伙伴展示柜

- **组件**：`<Page3Pixels />`
- **Canvas 实现**：
  - 划分为网格，每个格子绘制一个像素角色。
  - 每个角色独立动画：使用 `createSignal` 记录当前帧索引，通过 `setInterval` 驱动帧更新。
  - 悬停检测：通过 `onMouseMove` 计算所在格子，使用 GSAP 对角色进行“打招呼”动画（如跳跃）。
- **角色动画**：提供精灵表数据（或直接 Canvas 绘制简单角色）。帧率 8 fps。

### 第 4 页：焊接中的未来游戏

- **组件**：`<Page4Welding />`
- **单 Canvas 绘制**：
  - 分层绘制：背景工作台、平板、游戏角色、手、火花。
  - 游戏角色动画：使用 `createSignal` 控制帧，循环播放。
  - 手部控制：两只手的位置用信号存储，通过 GSAP 实现点击后的平滑移动。
- **点击交互**：
  - 监听 Canvas 点击，判断是否在平板区域。
  - 如果是，计算最近的手，使用 GSAP 移动手到点击坐标（`.to()` 动画）。
  - 到达后触发火花粒子：在 Canvas 上绘制多帧粒子（可用 `createEffect` 驱动粒子状态）。
  - 若点击在主角或怪物上，使用 GSAP 进行闪烁动画（改变透明度或颜色滤镜）。
- **焊接火花**：粒子系统简单实现，每帧更新位置和寿命，在动画循环中绘制。

### 3.5 游戏详情页（GameDetail）

- **组件**：`<GameDetail />`
- **功能**：根据路由参数 `id` 加载对应游戏的图片集（位于 `public/assets/images/games/[id]/` 下），并随机选择一种展示模式。模式有三种：**相册模式**、**3D轮播卡牌模式**、**抽签盒子模式**。随机规则：每个游戏独立记录上一次使用的模式，本次随机时排除该模式，确保不会连续两次相同（交替随机）。使用 `localStorage` 存储每个游戏的上次模式。
- **图片加载**：假设图片文件按数字序号命名（`1.jpg`, `2.jpg`, ...），或读取目录列表（前端无法直接读取目录，可在代码中预定义图片列表，或通过API模拟。此处简化：在组件中根据 `id` 定义图片数量，例如 `LOL` 有8张图，`荒野大镖客2` 有6张等。可在代码注释中说明需要手动配置图片列表）。

#### 模式 A：相册模式

- **视觉**：类似翻页相册，中央显示当前图片，左右两侧有箭头按钮。下方显示缩略图或页码。
- **交互**：
  - 点击左/右箭头或左右方向键切换图片，有平滑的滑动/翻页动画（可使用 GSAP 或 CSS transition）。
  - 支持循环切换。
  - 点击图片可查看大图（可选，暂不要求）。
- **实现**：使用 Solid 的信号 `currentIndex` 控制显示的图片。

#### 模式 B：3D 轮播卡牌模式

- **视觉**：卡片围绕垂直 Y 轴排列成一个圆环（跑马灯效果），每个卡片正面显示一张游戏图集中的图片。所有卡片面向圆心。
- **交互**：
  - **自动转动**：卡片会自动一格一格地转动（步进式，非连续转动），每次转动一个卡位的角度（例如有8张卡片，每次转动 360/8 度）。转动间隔可设为3秒。
  - **点击卡片**：点击任意卡片时，该卡片会以“抽卡动画”效果（如飞出、放大）移动到屏幕中央，显示为大图。其他卡片半透明或隐藏。
  - **退出详情**：大图右上角有一个“退出游戏”按钮，点击后大图以反向动画回到轮播原位，恢复轮播状态。
- **实现**：
  - 使用 CSS 3D 变换或 Canvas 绘制。推荐使用 CSS 3D + Solid 动态样式，因为卡片数量不多。
  - 卡片容器设置 `transform-style: preserve-3d`，每个卡片通过 `rotateY` 和 `translateZ` 定位。
  - 自动转动：使用 `setInterval` 更新旋转角度信号。
  - 点击卡片：记录选中卡片索引，隐藏其他卡片，将选中卡片通过 GSAP 动画移动到屏幕中央（可能需要临时改变其定位方式）。
  - 退出：恢复所有卡片状态，重新开始自动转动。

#### 模式 C：抽签盒子模式

- **视觉**：一个古朴的抽签盒子（可用 SVG 或 Canvas 绘制），盒子旁边有一个签筒或直接盒子本身可点击。
- **交互**：
  - **点击盒子**：盒子抖动（GSAP 震动动画），并随机“吐”出一个签（签面显示一张图片的局部裁剪）。签从盒子中弹出，落在盒子前方。
  - **点击签**：签以画轴展开的动画效果（从中间向两边展开，或从卷曲到平铺）展示完整图片。展开后图片充满屏幕大部分区域。
  - **关闭展开**：点击展开后的图片或外部区域，图片卷回收缩为签，回到盒子旁。
  - 再次点击盒子可抽取另一张签（图集中的下一张图片，循环）。
- **实现**：
  - 盒子抖动：使用 GSAP 的 `to` 动画在 `x` 和 `rotation` 上快速变化。
  - 签的弹出：可使用 CSS 或 Canvas 绘制签的形状，上面用 Canvas 裁剪图片的一部分（可用 `ctx.drawImage` 只绘制图片的某个矩形区域）。
  - 画轴展开：使用 GSAP 对签的宽度进行补间，同时改变图片显示区域，或使用两个卷轴元素向两边移动来揭示图片。

## 六、注意事项

- **滚动性能**：确保 Canvas 动画在滚动时暂停或降低帧率（使用 `IntersectionObserver` 判断是否在视口内）。
- **GSAP 清理**：组件卸载时需调用 `gsap.killTweensOf()` 或 `timeline.kill()` 避免内存泄漏。
- **Solid 响应式**：使用 `createEffect` 时注意避免无限循环，合理依赖信号。
- **Tailwind 自定义动画**：在 `tailwind.config.js` 中扩展 `keyframes` 和 `animation`。
- **随机模式交替**：在 `GameDetail` 组件中，通过 `localStorage` 存储每个游戏的上次模式键，例如 `lastMode_${id}`。本次随机从三种模式中排除上次模式，然后随机选择一种，并更新存储。
